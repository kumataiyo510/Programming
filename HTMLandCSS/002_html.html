Hyper Text Markup Language

HTMLの用語

  • 要素（Element）
    • 開始タグと終了タグに囲まれた範囲が「要素」です
    例）
    <タグ>要素<タグ>
  • 属性（Attribute）
    • 要素に対する様々なオプション設定のような役割を持つのが「属性」です
    例）
    <a href="http://www.yahoo.co.jp/">Yahoo! Japan</a>
    属性名属性値

HTMLの基本要素

  1. DOCTYPE宣言
    2. 使用するHTMLの種類を宣言する部分
  2. html要素
    3. HTML文書の最上位の（ルート）要素。headとbodyに分かれる
  3. head要素
    4. 文書タイトル・文字コード・キーワード・説明文・関連ファイルなどの文書の基本情報を記述する部分
    ブラウザには表示されない
  4. body要素
    5. コンテンツを記述する部分。ここに記述したものが実際にブラウザの画面に表示される
基本的にHTML文書は上記4要素にタグを入れ子構造（ネスト）にして作成する

html要素

html要素はHTML文書の最上位の要素であり、文書全体を包括する要素となります。html要素には一般的にlang属性を機j通するのが慣例となっています。
例）
<html lang"ja">

head要素

HTML文書のタイトル、文字コード、キーワード等、文書の補足情報を記載するのがhead要素です。
CSSやJavaScriptなどの外部読込ファイルの指定、検索エンジン向けの情報など、様々な情報を必要に応じて記述します。
例）
<head>～</head>

title要素

title要素はその名の通りHTML文書のタイトルを表します。

meta要素

meta要素は、文字コード・文書の概要・キーワードなど、ブラウザ画面には表示されない文書情報を記述するための要素です。
例）
<meta charset"utf-8">
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
<meta name="description" content="文書の概要が入ります">
<meta name="keywords" content"キーワードA、キーワードB">
※文字コード設定の際は、実際のHTMLファイルの文字コードと、meta要素の文字コードを一致させること

HTMLタグでマークアップする

  1. 見出しをマークアップする
    2. <h1>見出しテキスト</h1>
    見出しはh1からh6まで6段階用意されています。最上位の見出しがh1で、これはページに一つ必須となります。h2以下は文書構造に応じて適宜使用していきますが、
    間のレベルを飛ばしたり、レベルの上下関係を入れ替えたりすることは原則としてできません。

  2. 段落をマークアップする
    3. <p>段落テキスト</p>
    段落でマークアップされたところは自動的に改行されて前後に空白ができます。

  3. 箇条書きをマークアップする
    4. <ul>
    　　<li>箇条書き</li>
    </ul>
    段落でマークアップされたところは自動的に改行されて前後に空白ができます。
    箇条書きリストにはul要素とol要素が使えます。それぞれ順不同と順序を示す要素である。
    また、箇条書きタグは入れ子（ネスト）にして複雑な階層構造を持つ箇条書きリストを作ることも可能です。。

  4. 記述リスト（定義リスト）をマークアップする
    5. <dl>
    　　<dt>項目タイトル</dt>
    　　<d>項目内容テキスト</dd>
    </dl>
    記述リストとは、「項目とその説明」がワンセットになったリスト構造です。

  5. 情報のグループ化
    6. <div>ブロック領域</div>
    「見出し」「段落」といった文章構造としての意味はないが、ソースコードをグループ化するという意味で使用します。
    div要素にはその役割に応じたわかりやすい名前を付けて管理をしておきます。要素の名前はid属性を使って指定しておきます。

  6. divをsectionに置き換える
    7. <section>セクション領域</section>
    見出しとそのコンテンツといった文書構造的に意味のあるグループを指定する要素タグである。

入れ子（ネスト）

HTMLでは要素同士を入れ子にして組み上げていくのですが、「ある要素の中にどんな要素を入れることができるか」を定めたルールがありそのルールを「コンテンツモデル」と呼びます。
コンテンツモデルは「ブロック要素」と「インライン要素」という考え方を使用する考え方です。
ブロック要素の中にインライン要素を入れることはできますが、インライン要素の中にブロック要素を入れることはできないという考え方です。
ちなみに手軽にブロック・インラインを見分ける方法は、その要素を記述した時にブラウザ側が自動的に改行をするかどうかということです。改行されるものはブロックレベル、改行されないものはインラインレベルとなります。

HTML文書を見やすく整理する

  1. <br>
    2. 強制改行、このように開始タグのみで終了タグが存在しない要素を「空要素」と呼びます。
  2. <strong>重要な語句</strong>
    3. 重要な語句がある場合、strong要素を使って強調できます。単にデザイン的に太字で見せたいという理由でstrong要素を使用してはいけません。
    ※em要素はアクセントをつけて強調するという意味で、重要性を示す意味はありません。
  3. <img src="①画像ファイルへのパス" width="②サイズ" height="②サイズ" alt="③代替テキスト">
  4. <a href="①リンク先情報">コンテンツ</a>
    • ページ内リンクの場合 #id名
    • サイト内別ページへのリンクの場合 パス名
    • 外部サイトへのリンクの場合 URL
      • ※target属性をblankにすると新規ウィンドウタブで開ける
    • メールアドレスにリンクを貼る mailto:メールアドレスと記述
    • 電話番号にリンクを貼る tel:電話番号を記述
  5. 著作権表記であることを明示する
    6. <small>テキストテキスト</small>

フォームを設置する
form_test_header form_input_area

<form id="①フォーム名" action="②データ送信先のパス" method="③データ送信方法"></form>
①id 属性：どこのフォームから送られてきたのかを判別するために使用する名前
②action 属性：データ送信先（主に Web サーバに用意されたプログラムファイル）のパス
③method 属性：データ送信方式。get （データを URL の一部として送信）または post （データを本文として送信）のどちらかを選択

その他の属性

  • autofocus 属性
    • 指定した入力フォームに自動的にカーソルをフォーカスさせることができます
  • autocomplete 属性
    • 以前入力した内容に基づいて自動的に入力候補を補完する機能です
  • placeholder 属性
    • 入力フォームの中に短いヒントを表示させて入力の手助けを行います
  • required 属性
    • 必須項目を設定することができます
  • min max step 属性
    • 入力の際の、最小・最大・ステップ値を設定することができます
